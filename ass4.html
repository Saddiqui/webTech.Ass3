Appendix A: Web content Accessibility guidelines
Guideline evaluation
1.1.1 Non-text Content: All non-text content that is presented to the user has a text alternative that serves the equivalent purpose, except for the situations listed below. (Level A) The non-text content in our webpage doesnt have any alternative that serves the equivalent purpose. The links that are on the homepage, cannot be easily accessed in forms that could be needed by others. For example any blind person may not be able to easily detect the location of a link, but with the help of a form that supports spoken words (speech) this can be easily Accessible. Also the navigation bar and the tables do not have a text-alternative that serve the equivalent purpose. This can be compared to the example above. In the case of the navigation bar and the table, no other form is supported on our webpage for easy Accessibility. We can conclude that our homepage does not have any text alternatives that serve the equivalent purpose as non-text content. Expect for the image and formatting content, whose only purpose is for decoration.
1.3.1 Info and Relationships: Information, structure, and relationships conveyed through presentation can be programmatically determined or are available in text. (Level A) Information, structure and relationships on the homepage are conveyed programmatically on the homepage. Information that is provided on the homepage like text, table and images are all styled in css by giving the corresponding texts the same color, font or even emphasise. For example: the texts in the navigationbar are all the same color as well as words that need to stand out. Also information that needs to stand out is emphasized by using bold or colored text. For example: the text in the navigationbar is all the same color for every navigationlink. This lets the user know that the links in the navigationbar have the same relation.(to navigate to the corrresponding page) Different heading are used in the homepage, but could have been used in a more proper manner if there was more text information on the homepage. The structure between the different products is also done programmaticly by the use of a table. Like the first row in the tables, they are written in bold letters. So the user can easily see the structure of the table, with the first row serving the purpose of naming the different columns. And the rows going vertically for each column name with the corresponding information on the product. Also giving the table a caption contributes to giving the user extra information for a userfriendly experience, as the user can easily find the subject of the table. lastly the input fiels that are required, dont show they are required until you try to subit the input. This however can be done easily by the use of a astrisk(*) character at the end of the fieldname in the form.
1.3.2 Meaningful Sequence: When the sequence in which content is presented affects its meaning, a correct reading sequence can be programmatically determined. (Level A) The homepage has a meaningful sequence to some extend. The title(header) of the homepage is provided on top of the page. Following this there is a navigationbar, that if positioned in another sequence would not be userfriendly. After the navigationbar there is some text and then come two tables. and the website has a footer at the end. For the sake of having a basic html webpage, the sequence is correct and does not deprive the webpage of it's meaning.
1.3.3 Sensory Characteristics: Instructions provided for understanding and operating content do not rely solely on sensory characteristics of components such as shape, size, visual location, orientation, or sound. (Level A) We dont think this guideline applies to our webpage. As we do not have any instructions or operating content that is not accessible by text only. For example: someone who is colorblind or does have a weaker eyesight can still use the webpage as intended Without having difficulty understanding the operating content of instructions.
1.4.1 Use of Color: Color is not used as the only visual means of conveying information, indicating an action, prompting a response, or distinguishing a visual element. (Level A) The only color distinguished element on the page that can be hard to convey to users that do not perceive visuals as intentended, are the links under the navigation bar. These are colored different for each of the uses like unvisited links or visited links. These could for example be positioned in a ordered list, the unvisited links list and the visited links list. Also the hover collor (light grey) on the navigationbar, may not be recognised by some visual impaired users. However this shouldnt make a big difference in the Accessibility of the homepage.
1.4.2 Audio Control: If any audio on a Web page plays automatically for more than 3 seconds, either a mechanism is available to pause or stop the audio, or a mechanism is available to control audio volume independently from the overall system volume level. (Level A) this does not apply to our webpage, as there is no audio being played on the webpage.
2.1.1 Keyboard: All functionality of the content is operable through a keyboard interface without requiring specific timings for individual keystrokes, except where the underlying function requires input that depends on the path of the user's movement and not just the endpoints. (Level A) The webpage is accessible with any keyboard. There is no requirement for specific timings for individual keystrokes. The user can access any keyboard that is connected to the webpage and browse the webpage, as well as use the functions on the webpage. Only moving the cursor, typing, clicking, sizing and selecting are accessible on the webpage. However hand writing, painting, etc is not accessible on the webpage. There is no specific feature or program on the webpage that allows this. Thus, not succesfully complying to this guideline.
2.1.2 No Keyboard Trap: If keyboard focus can be moved to a component of the page using a keyboard interface, then focus can be moved away from that component using only a keyboard interface, and, if it requires more than unmodified arrow or tab keys or other standard exit methods, the user is advised of the method for moving focus away. (Level A) This does not apply to our webpage, as we do not have any interface in which the keyboard could be trapped.
2.4.1 Bypass Blocks: A mechanism is available to bypass blocks of content that are repeated on multiple Web pages. (Level A) There is no mechanism available to bypass blocks of content. This is due to no other webpages being made, expect for the homepage. So this guideline does not apply to our assignment.
2.4.2 Page Titled: Web pages have titles that describe topic or purpose. (Level A) In this case the title of the webpage is already named to the appropriate name "assigntment 1: homepage of a webshop". This indicating what the subject/intend for this webpage is. However there aren't sufficient webpages to show this impemented correctly, as we only made one webpage.
2.4.3 Focus Order: If a Web page can be navigated sequentially and the navigation sequences affect meaning or operation, focusable components receive focus in an order that preserves meaning and operability. (Level A) The webpage can be navigated sequentially. However the webpage is made up from required elements from assignment 1 and 2, that do not require us to give the webpage a specific meaning when navigated sequentially. The webpage does have a basic sequence and follows the left hand navigation with the corresponding css stylesheet. The webpage can be operated easily with a logical tab order. For example: the navigation bar and the form.
2.4.4 Link Purpose (In Context): The purpose of each link can be determined from the link text alone or from the link text together with its programmatically determined link context, except where the purpose of the link would be ambiguous to users in general. (Level A) This guideline does not apply to our webpage, as we do not have any links on the webpage that are in text links.
